# Frontend Guide

이 문서는 현재 프런트엔드의 사용자 화면 구조를 설명합니다. 기준은 React + TypeScript + Vite 단일 페이지 앱입니다.

핵심 원칙은 하나입니다.

- 기능 이름과 실제 역할이 일치해야 합니다.
- 내부 구현 단계보다 사용자가 이해하는 작업 흐름을 먼저 보여줘야 합니다.
- 최근 생성 이력은 여기저기 흩뿌리지 않고 한 화면에서 관리해야 합니다.

## 역할 요약

프런트엔드는 아래를 담당합니다.

- 메인 TTS 화면 제공
- 참조 음성 기반 스타일 추출과 저장
- 설명문 기반 스타일 설계와 저장
- 저장한 프리셋의 프로젝트 단위 반복 생성
- 장문 대본 생성
- 갤러리형 생성 이력 관리
- 데이터셋 생성과 학습 실행 분리
- 사운드 효과, 보이스 체인저, 오디오 분리 같은 오디오 도구 페이지 제공

## 현재 페이지 구조

좌측 네비게이션 기준으로 페이지를 나눕니다.

- `홈`
- `갤러리`
- `텍스트 음성 변환`
- `목소리 복제`
- `목소리 설계`
- `프리셋 프로젝트`
- `스토리 스튜디오`
- `사운드 효과`
- `보이스 체인저`
- `오디오 분리`
- `데이터셋 만들기`
- `훈련 랩`

중요한 점:

- 예전 `빠르게 들어보기`와 `모델 선택형 추론`은 `텍스트 음성 변환`으로 합칩니다.
- `목소리 복제`와 `목소리 설계`는 분리합니다.
- `스토리 스튜디오`는 장시간 대본용입니다.
- `최근 생성 이력`은 여러 화면에 반복 노출하지 않고 `갤러리`에 모읍니다.
- `데이터셋 만들기`와 `훈련 랩`은 분리합니다.

## 초기 로딩 구조

프런트는 시작 시 `GET /api/bootstrap`을 호출해 초기 상태를 한 번에 읽습니다.

대표 항목:

- `health`
- `models`
- `speakers`
- `history`
- `presets`
- `datasets`
- `finetune_runs`
- `audio_assets`

초기 화면은 bootstrap 응답을 기준으로 채우고, 각 화면에서 필요한 후속 요청만 추가합니다.

## 페이지별 역할

### 홈

- 지금 바로 할 수 있는 작업 진입점
- 모델/데이터셋/훈련 준비 상태 요약
- 텍스트 음성 변환, 목소리 복제, 데이터셋 만들기 같은 핵심 기능으로 이동

### 갤러리

- 최근 생성 이력 관리 전용 화면
- 생성 음성, 저장 프리셋, 프로젝트 자산, 학습 결과 자산을 한곳에서 탐색
- 다른 화면에는 “최근 생성 이력” 카드를 반복 배치하지 않는 것이 원칙

### 텍스트 음성 변환

- 메인 TTS 화면
- 짧게 들어보기와 모델 선택형 추론을 한곳에서 처리
- 자주 쓰는 stock `CustomVoice`는 빠르게 선택 가능
- 학습된 최종 모델도 여기서 선택 가능
- `seed`, `top_k`, `top_p`, `temperature` 같은 항목은 기본 화면이 아니라 `고급 제어`에 넣습니다

사용자 관점에서의 모델 차이:

- `CustomVoice`
  화자와 말투 지시를 바로 받을 수 있어서 가장 쉽게 쓸 수 있습니다.
- `Base`
  누구 목소리로 말할지 먼저 알려줘야 하므로, 참조 음성이나 저장된 프리셋이 필요합니다.

즉 `Base`가 참조 음성을 요구하는 이유는 “음색 기준을 먼저 넣어야 하는 모델”이기 때문입니다.

### 목소리 복제

- 참조 음성에서 clone prompt를 만들고 저장하는 화면
- `Base` 모델을 사용합니다
- 업로드 음성 또는 서버 자산을 참조 음성으로 고를 수 있습니다
- 만든 결과는 나중에 `텍스트 음성 변환`, `프리셋 프로젝트`, `스토리 스튜디오`에서 재사용합니다

### 목소리 설계

- 설명문으로 스타일을 만드는 화면
- `VoiceDesign` 모델을 사용합니다
- 생성된 스타일은 프리셋으로 저장할 수 있어야 합니다
- “복제”와 “설계”는 다른 작업이므로 같은 탭에 섞지 않습니다

### 프리셋 프로젝트

- 저장된 프리셋을 반복 생성용 프로젝트로 관리하는 화면
- 한 프로젝트 안에서 대사, 말투 지시, 언어, 생성 옵션을 바꿔가며 여러 번 생성합니다
- clone prompt 저장과 voice design 저장 이후의 재활용 중심 화면입니다

### 스토리 스튜디오

- 장문 생성 전용 화면
- 긴 대본을 한 번에 읽게 하는 것이 목적입니다
- `장면 1`, `장면 2` 같은 메타 텍스트를 읽게 만드는 기능이 아닙니다
- 멀티 화자 스크립트 데이터 생성은 별도 기능으로 분리하는 것이 맞습니다

### 사운드 효과

- 효과음 생성 전용 화면
- `MMAudio` 준비 상태에 따라 사용 가능 여부가 달라집니다

### 보이스 체인저

- 기존 음성 파일의 보컬 음색을 바꾸는 audio-to-audio 화면
- `Applio / RVC` 계열 모델을 전제로 합니다
- 전사 후 다시 읽는 TTS 보조 기능과는 다른 기능입니다

### 오디오 분리

- 업로드 또는 서버 자산을 선택해 분리 작업을 실행하는 화면

### 데이터셋 만들기

- 학습용 오디오와 텍스트를 정리하는 화면
- 샘플 행을 추가하고, 필요한 경우 Whisper 자동 전사를 사용합니다
- 결과는 `data/datasets/<dataset_id>/` 아래에 오디오와 JSONL이 함께 정리됩니다

이 화면에서 사용자에게 명확히 보여줘야 하는 것:

- 샘플 권장 수
- 기준 음성이 무엇인지
- 오디오만 넣으면 자동 전사가 어떻게 적용되는지
- 데이터셋이 만들어지면 바로 학습에 쓸 수 있는지

### 훈련 랩

- 이미 준비된 데이터셋을 골라 학습을 시작하는 화면
- 데이터셋 생성 화면과 분리합니다
- `Base Fine-Tune`과 `CustomVoice Fine-Tune` 중 하나를 고릅니다
- 여러 epoch 체크포인트를 모두 사용자에게 노출하는 대신, 최종 선택 모델 하나를 다시 쓰는 방향으로 안내합니다

## 생성과 기록 UX 원칙

- 최근 생성 이력은 `갤러리` 한 탭에 모읍니다
- 프리셋은 `목소리 복제`, `목소리 설계`에서 저장하고, 반복 생성은 `프리셋 프로젝트`에서 합니다
- 학습 결과는 “훈련 중간 체크포인트”보다 “최종 선택 모델” 중심으로 보여주는 편이 사용자에게 더 이해하기 쉽습니다
- 하이퍼파라미터는 기본 화면을 복잡하게 만들지 않도록 `고급 제어` 안에 넣습니다

## 학습 관련 사용자 안내 문구

프런트 문구는 아래 기준을 따르는 것이 좋습니다.

- `1~5개 샘플`
  파이프라인 점검용
- `10개 안팎`
  아주 작은 실험용
- `20~50개`
  최소한의 화자 적응을 기대할 수 있는 구간
- `50개 이상`
  음색 반영과 안정성이 더 나아질 가능성이 큼

또한 아래 기대치를 함께 적어야 합니다.

- `Base Fine-Tune`
  음색 적응 실험에는 쓸 수 있지만, instruct 준수까지 자동으로 좋아지는 것은 아닙니다.
- `CustomVoice Fine-Tune`
  말투 지시를 유지한 채 dataset 음색을 반영하는 후보 경로입니다. 다만 데이터 품질과 양에 따라 차이가 큽니다.
