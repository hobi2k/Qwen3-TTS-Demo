# Qwen3-TTS-Demo Style Guide

## 문서화 원칙

- 모든 Python 함수와 메서드는 triple-quoted docstring을 사용한다.
- docstring 첫 줄에는 함수가 무엇을 하는지 한 문장으로 적는다.
- 인자가 있으면 `Args:` 섹션을 추가한다.
- 반환값이 있으면 `Returns:` 섹션을 추가한다.
- 인자와 반환값이 모두 없으면 함수 동작 설명만 남기고 `Args:`, `Returns:`는 생략한다.
- 비동기 함수, 프로퍼티, 내부 헬퍼도 같은 규칙을 따른다.

예시:

```python
def build_message(name: str) -> str:
    """사용자 이름을 포함한 환영 메시지를 만든다.

    Args:
        name: 화면에 표시할 사용자 이름.

    Returns:
        포맷팅된 환영 문장.
    """
```

## 주석 원칙

- 주석은 코드가 "무엇을" 하는지보다 "왜 이렇게 처리하는지"를 설명한다.
- 모델 추론, 시뮬레이션 분기, 파일 저장 형식, 외부 프로세스 실행처럼 의도가 중요한 구간에는 상세 주석을 남긴다.
- 자명한 한 줄짜리 코드에는 주석을 달지 않는다.
- 오래된 구현 맥락이나 추측성 설명은 남기지 않는다.

## Python 스타일 원칙

- 중복 로직은 헬퍼 함수로 묶어 API 핸들러를 짧고 읽기 쉽게 유지한다.
- 경로는 `pathlib.Path` 중심으로 다루고, 저장 시점에만 문자열로 변환한다.
- JSON 저장 구조는 한 함수에서 일관되게 생성하고, 응답 모델 변환은 반환 직전에 수행한다.
- 예외 메시지는 API 사용자가 바로 조치할 수 있게 구체적으로 작성한다.
- 시뮬레이션 모드와 실제 모델 모드는 같은 데이터 구조를 반환하도록 유지한다.

## 프로젝트 운영 원칙

- 루트 `pyproject.toml`은 `uv` 기준으로 의존성을 관리한다.
- 업스트림 `Qwen3-TTS` 패키지는 로컬 editable source로 연결한다.
- 새 의존성을 추가할 때는 실제 사용 파일과 설치 목적을 함께 문서화한다.
